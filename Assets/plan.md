# 가위바위보 게임 — 개발 추천 콘텐츠

강의용·프로토타입 수준을 넘기지 않으면서도 체감이 좋아지는 방향 위주로 정리했습니다.

## 게임플레이

- **라운드 수 / 목표 점수 설정**: 3선승, 7점까지 등 난이도·플레이 타임 조절.
- **무승부 처리**: 무승부 시 점수 변동 없음 유지 + 연속 무승부 카운트, 또는 “한 번 더” 연출.
- **콤보 / 연승 표시**: 연속 승리 시 짧은 피드백(텍스트·색·사운드)으로 몰입감 보강.
- **제한 시간**: 선택까지 타이머; 시간 초과 시 랜덤 선택 또는 패배 처리.
- **가중 랜덤 AI**: 단순 균등 랜덤 대신, 최근 플레이어 패턴을 살짝 반영한 “약한 인간형” 컴퓨터.

## 연출·피드백

- **승패 짧은 애니메이션**: 손 이미지 트윈, 파티클, 카메라 살짝 흔들림.
- **사운드**: 버튼 클릭, 승·패·무승부, 게임 종료 각각 다른 효과음.
- **접근성**: 색만으로 구분되지 않도록 아이콘/라벨 보강, 버튼 포커스(키보드·패드) 지원.

## UI·메타

- **설정 패널**: BGM/효과음 볼륨, 전체화면, 목표 점수.
- **통계 화면**: 총 판수, 승률, 가장 많이 낸 손(가위/바위/보 비율).
- **튜토리얼 / 첫 실행 안내**: 한 화면짜리 규칙 설명 또는 스킵 가능 팝업.

## 기술·구조 (과제·포트폴리오용)

- **데이터 분리**: 점수·설정을 ScriptableObject 또는 간단한 JSON으로 분리해 Inspector 의존도 낮추기.
- **상태 머신**: 대기 → 선택 → 결과 → 종료 같은 흐름을 enum/상태로 명시하면 버그·중복 입력 방지에 유리.
- **로컬라이제이션**: 한/영 문자열 테이블만 분리해도 확장성이 좋아짐.

## 멀티·확장 (선택)

- **로컬 2P**: 같은 화면에서 턴제 또는 동시 입력(숨김 선택 후 공개).
- **간단 네트워크**: 수업 범위를 넘길 수 있으나, “동기화되는 점수만” 수준의 초경량 P2P는 학습 주제로 의미 있음.
